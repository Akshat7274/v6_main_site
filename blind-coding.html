  • Blind Coding

The programmer will work on a code without running it until completion. Usually a terrible idea, but it makes for a fun competition!


Rules and Regulations

  • This is an individual event.
  • This event consists of two rounds.
  • Each participant has to write the code for the problem statement, with the MONITOR OFF.
  • In round 1, program statements of medium difficulty will be given.
  • In round 2, program statements of high difficulty will be given.
  • Cheating, using a phone or any other gadgets, will result in instant disqualification of the participant.

Judgement Criteria

  • Participants whose code gets executed will be given a higher preference
  • The second preference will be given to code with the least no. of errors
  • In case of tie-in no. of errors then the timing will be considered
  • Shortlisted participants of round 1 will appear for round 2 for which the above rules will be applicable.
